Task Description

Make PL/Perl6 (a part of PL/Parrot) work on the latest Rakudo Perl 6 release.

Steps To Complete This Task

Recently, a change in Rakudo Perl 6 and/or Parrot caused PL/Perl6 to start failing some tests. This task to identify the problem and fix it. It is probably very little code that needs to change, but finding out what the problem is will be challenging. Currently, when you run the PL/Perl6 test suite on a Rakudo Star release, or the Rakudo master branch, the test suite goes into an infinite loop. More specifically, the PostgreSQL process that the test suite spawns starts to take 100% CPU and is eventually killed by the postmaster process, which controls all the child PostgreSQL processes. This task will require identifying exactly which part of the code in plparrot.c is causing the behaviour. Most likely, PL/Parrot is getting something from Rakudo which is invalid, and then passes it along to PostgreSQL, which doesn't react well.
  1. Create a fork of plparrot.git on github.com
  2. Read the PL/Parrot README
  3. Perform all the necessary code and/or documentation changes
  4. Build parrot and run it's test suite (make test_plperl6) to verify that things still work
  5. Create a Github pull request (button on the upper right of your fork) to have your changes incorporated into the master repository

Benefits

  1. This will allow PL/Perl6 to continue to be developed and allow people to write database stored procedures in Perl 6!

Requirements

  1. Knowledge of C and printf
  2. Some knowledge of PIR and Rakudo Perl 6
  3. Abillity to debug.
